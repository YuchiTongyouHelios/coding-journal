# Month 1: February 14 – March 14, 2026

## The Beginning
I started learning to code on February 14, 2026, with zero experience. This log captures everything I built, learned, and struggled with in my first 30 days.

## Weekly Breakdown

### Week 1: Number Guesser
- **Project**: Command-line game with difficulty levels, high scores, and file storage.
- **Key Concepts**: Variables, `input()`, `random`, `if/else`, `while` loops, `try/except`, file I/O.
- **Takeaway**: Even simple games need planning. File saving taught me about persistence.

### Week 2: To-Do List App
- **Project**: Task manager with add/view/remove/complete, search, and file storage.
- **Key Concepts**: Lists, list methods (`.append`, `.remove`), functions, parameters, `enumerate`, search logic.
- **Takeaway**: Functions make code reusable and readable. Auto-save vs manual save is a design tradeoff.

### Week 3: Drilling Fundamentals
- **Focus**: Indexes, `return` vs `print`, function parameters, file save timing, list operations.
- **Key Insights**:
  - Indexes start at `0`, `-1` is last.
  - `print()` shows output, `return` sends data back.
  - Functions should return data; main code prints.
  - Small programs = same building blocks as big ones.

### Week 4: Building with New Concepts
- **Day 22**: Set up VS Code + Git, learned commit/push workflow.
- **Day 23**: Mad Libs game – practiced functions and user input.
- **Day 24**: Sentence Builder – lists, loops, `.join()`, `\n`.
- **Day 25**: Word Counter – dictionaries, counting words, most common word.
- **Day 26**: AI Violin Coach (with help) – learned about audio processing, but also learned a hard lesson about copy-pasting.
- **Day 27**: Tutorial day – reflected and planned next steps.

## The Honest Truth
On Day 26, I used AI to generate a complex violin pitch detector and copy-pasted it without understanding. I felt guilty and wrote a [public confession](Confession.md). That moment changed how I learn:
- **No more copy-paste without understanding every line.**
- **AI is a teacher, not a writer.**
- **Public accountability keeps me honest.**

I now follow the **Helios Protocol**:
1. Generate code with AI (if needed).
2. Explain each line in my own words.
3. Rewrite it manually in VS Code.
4. Commit only after understanding.
